# RNLI Premier League Predictor

An invite-only Premier League score prediction game for RNLI colleagues. Players predict the scoreline of every fixture before kickoff. Points are awarded for accuracy, and a live leaderboard tracks standings across the season.

**Live:** https://rnli-premierleague-predictor.vercel.app

---

## Stack

| Layer | Technology | Hosting |
|-------|-----------|---------|
| Frontend | React 19 + Vite + Tailwind CSS | Vercel |
| Backend | FastAPI (Python 3.12) | Render |
| Database | PostgreSQL | Supabase |

---

## Features

### For players
- **Predictions** — predict the scoreline of every fixture before kickoff; predictions lock automatically at kickoff time
- **Wildcard** — activate once per season to double your points for a chosen gameweek
- **Leaderboard** — overall standings with rank deltas, form dots, and a Recharts points-over-time area chart
- **Fixtures** — full fixture list with live status badges (upcoming / starting soon / live / awaiting result / completed)
- **Dashboard** — personal stats: total points, accuracy %, best week, prediction breakdown chart
- **Invite-only** — players register via a single-use invite link generated by an admin

### For admins
- **Admin panel** with 7 tabs: Overview · Users · Wildcards · Predictions · Missing · Fixtures · Invites
- Upload fixtures via CSV (upsert — safe to re-run, preserves predictions and results)
- Enter match results per gameweek (scoring calculates automatically)
- Postpone / reschedule individual fixtures
- Manage users: view all accounts, reset passwords, promote to admin
- Generate and revoke single-use invite links
- See who hasn't submitted predictions yet (Missing tab)
- **Results entry** — dedicated admin page for entering full-time scores

### Scoring
| Outcome | Points |
|---------|--------|
| Exact score | 5 |
| Correct result (right winner / draw) | 2 |
| Wrong | 0 |

Wildcard multiplies all points for the activated gameweek by ×2.

## Design System

All pages share an editorial RNLI navy + Premier League gold design language:

- **Colours:** `#003087` (RNLI navy) · `#FFB81C` (PL gold) · dark mode default
- **CSS namespace pattern:** each page uses a prefixed namespace (`home-*`, `lb2-*`, `fx-*`, `pd-*`, `db-*`, `adm-*`, `lgn-*`) appended to `index.css` — never modifying existing classes
- **Tailwind custom colours:** `rnli-blue`, `rnli-blue-light`, `rnli-blue-dark`, `rnli-yellow`, `rnli-yellow-light`, `rnli-yellow-dark`

## Local Development

### Prerequisites
- Python 3.12+
- Node.js 18+
- A Supabase project (or use the existing one with `.env` from a team member)

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Copy and fill in environment variables
cp .env.example .env           # or create .env manually (see below)

uvicorn main:app --reload --port 8000
```

**Backend `.env`**
```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
SECRET_KEY=your-jwt-secret-key
ALLOWED_ORIGINS=http://localhost:5173
```

API docs available at http://localhost:8000/docs once running.

### Frontend

```bash
cd frontend
npm install

# Create .env.local
echo "VITE_API_URL=http://localhost:8000" > .env.local

npm run dev
```

Opens at http://localhost:5173.

## Deployment

### Frontend → Vercel

Deployments are triggered manually via CLI (GitHub auto-deploy was disabled due to stale caching):

```bash
cd rnli-premierleague-predictor   # repo root
npx vercel deploy --prod
```

Set `VITE_API_URL` in the Vercel project environment variables to point at the Render backend URL.

### Backend → Render

The backend auto-deploys from the `main` branch on Render. Push to `main` and Render picks it up within ~2 minutes.

`backend/render.yaml` configures the service. Required Render environment variables:

```
SUPABASE_URL
SUPABASE_KEY
SUPABASE_SERVICE_ROLE_KEY
SECRET_KEY
ALLOWED_ORIGINS=https://rnli-premierleague-predictor.vercel.app
```

### Database → Supabase

Supabase is managed via the Supabase dashboard. Run `backend/migrate.py` to apply any schema changes:

```bash
cd backend
python migrate.py
```

## Creating the First Admin User

After a fresh database setup:

```bash
cd backend
python scripts/create_admin.py
```

Follow the prompts to set username, email and password. The account is created with the `admin` role. Subsequent admins can be promoted via the Admin Panel → Users tab.

## Uploading Fixtures

**Via Admin Panel (recommended):** Admin Panel → Fixtures → drag and drop a CSV file.

**Via CLI:**
```bash
cd backend
python import_fixtures.py path/to/fixtures.csv
```

CSV format — see `fixtures.csv` in the repo root for an example:

```csv
week,date,home,away,time
1,2025-08-16,Arsenal,Chelsea,12:30
1,2025-08-16,Liverpool,Everton,15:00
```

`time` is optional but enables automatic prediction locking at kickoff.
